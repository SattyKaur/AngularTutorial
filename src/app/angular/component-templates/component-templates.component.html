Components & Templates

Components, templates and modules are what make up an Angular app. Every app has at least one, many big applications use hundreds of components and modules.

What is a component?

Components are files that encapsulate data and logic, and displays a view through the HTML for that component. A single view can be made up of several components. E.g. you may have a component for the navigation bar, and a separate one for a media video, and another for a text body. The separation of components make them reusable and maintainable.

You can create a component in the CLI using the following command followed by the name of the component: 

    ng generate component home

This will generate 4 files:

  • css: In this file you can add styles you want to apply to this component.
  • ts: A Typescript file, in here you will perform any logic, functions, store or retrieve the data.
  • html: In this file you will write all the HTML which can be bounded to data in the typescript file.
  • spec.ts: This file contains unit tests for your component.

When you create a component need to add it to the list of declarations array in the module. (See Modules for more information) 

    app.module.ts
    import {{'{'}} HomeComponent } from './home.component';

    @NgModule ({{'{'}}
      declaration: [ HomeComponent ],
        imports: [  ],
        providers: [ ],
        bootstrap: [ AppComponent ]
    })
    export class AppModule {{'{'}} }

Inside our component typescript we need to add the @Component() decorator. This lets angular know it is a component. The decorator takes some metadata, here is an example of some of the most common properties.

  • selector: This is the tag name that we use to when we want to use the component elsewhere in the application E.g. <home-component></home-component>
  • template / templateUrl: You can add your html here inside the component or reference the html file associated to this component. This is required where as the other properties are not.
  • styles / stylesUrl: Here you can add any styles or reference a CSS file. It takes an array so you can have multiple CSS files
  • providers: Here you reference any services your component may use.
  • animations: Here you can add any animations you want for this component.
    home.component.ts

    @Component({{'{'}}
      selector: 'home-component',
        templateUrl: './home.component.html',
        stylesUrl: './home.component.css'
    })

    export class HomeComponent {{'{'}}  }
Templates

A template contains HTML code linked to its component file. A template is made up of basic HTML elements and also some Angular template syntax like below.

    home.component.ts
    title = 'Angular Home Page';

    home.component.html
    <h1>{{ "{{ title }\}" }}</h1>

You can also embed whole templates of other component inside a each other - creating a hierarchical system of templates. For example below we have a parent component template (home.component) with a child component inside (content). 

    home.component.html
    <h1>Homepage</h1>
    <content></content>

    content.component.html
    <p>Welcome to my page....</p>

Life Cycle Hooks

Each component has a life cycle, from when it is created, loaded, updated and then destroyed. Developers can tap into any of these life cycles hooks by implementing their interface. Each interface as a method named after the life cycle prefixed with ng. After the component is created, Angular calls the following life cycle hooks in this specific order.

  • ngOnChanges

    • Triggered each time there is a change in any input properties

  • ngOnInit

    • Called only once when the component has been initialized. You would initialize any data properties here instead of the constructor

  • ngDoCheck

    • Allows us to detect and act on changes that Angular wont detect on its own

  • ngAfterContentInit

    • Invoked after content projection (ng-content) has been initialized

  • ngAfterContentChecked

    • Each time a change in the content is detected through change detection

  • ngAfterViewInit

    • Invoked when the components view has fully initialized

  • ngAfterViewChecked

    • Invoked each time the view has been checked by change detection

  • ngOnDestroy

    • This method is invoked just before the component is destroyed
